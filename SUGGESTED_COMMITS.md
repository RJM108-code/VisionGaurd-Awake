# Suggested Git Commit Sequence

To make your GitHub repository look like a real development process, do **not** upload everything in one single commit.
Use a sequence like this:

1. `init: create project structure and add requirements`
2. `feat: add webcam capture and project configuration`
3. `feat: implement facial landmark based drowsiness detector`
4. `feat: add yawn detection, alerts, and event logging`
5. `docs: write complete README and usage instructions`
6. `docs: add BYOP project report`
7. `chore: add gitignore and clean project structure`

You can even make a few small edits between commits, such as:
- changing threshold values
- improving overlay text
- adding snapshot support
- updating README screenshots or explanation

This will satisfy the "actual development process" expectation much better than one final upload.
